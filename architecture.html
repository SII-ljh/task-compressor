Forward Pass

1
Encode — 长文本 context 通过 Qwen + LoRA 编码，得到 hidden states (B, Lc, D)
2
Prompt Encode — np=16 个可学习 latent tokens 对 frozen prompt embeddings 做 cross-attention，得到 prompt latents (B, np, D)
3
Compress — 将 prompt latents 与 nc=48 个 context tokens 拼接为 query (B, k, D)，经过 Perceiver 压缩 encoder hidden states → k=64 个 soft prompts
4
Decode — Frozen Qwen（LoRA 关闭）接收 [soft prompts | sep | prompt | response]，teacher-forcing 生成

LoRA Encoder Shared Qwen

在 Qwen 基础模型上添加 LoRA adapter，仅训练 ~1% 参数，编码长文本 context。

  • Target modules: q_proj, v_proj
  • LoRA rank: 128, alpha: 256
  • Dropout: 0.05

Prompt Encoder

将变长 user prompt 压缩为固定数量的 latent vectors，提供 task-aware 的 query 信号。

  • n_p=16 个 learnable latent tokens
  • Single cross-attention layer (Pre-LN + residual)
  • KV = frozen Qwen embedding(prompt)

Query-Conditioned Perceiver

核心压缩模块。Query 由 prompt latents + context tokens 组成，从 encoder hidden states 中提取信息。

Cross-Attention — query attends to encoder hidden states
Self-Attention — query tokens 之间交互
FFN — GELU activation, Pre-LN + residual

num_layers=2，输出 final LayerNorm + scale matching

Frozen Decoder Shared Qwen

与 Encoder 共享同一个 Qwen 实例，通过 toggle LoRA on/off 切换角色。Decoder 阶段 LoRA 关闭。

  • 输入: [compressed | sep | prompt | response]
  • 只对 response positions 计算 logits（节省 VRAM）
  • CE loss on response tokens (fp32)

Key Design Decisions

1
单 Qwen 实例 — Encoder 和 Decoder 共享权重，通过 PEFT 的 LoRA enable/disable 切换。节省显存，避免加载两份模型。
2
单阶段训练 — LoRA 初始化 delta ≈ 0，Perceiver 从一开始就能获得高质量的 Qwen hidden states，无需预训练。
3
差异化学习率 — Perceiver / Prompt Encoder 使用 5e-5，LoRA 使用 1e-5，平衡模块间的训练速度。
4
Scale Matching — Perceiver 输出通过 LayerNorm gamma 初始化为 embedding std，避免 decoder 输入 scale 不匹配导致 NaN。
